# Agentic PR Forensics

Use this skill to reason about PRs generated by coding agents.

## Forensic model

Audit the PR as a chain:

```text
spec -> agent/harness -> diff -> tests -> telemetry -> risk -> decision
```

## Evidence to collect

- Original task intent: issue, PR body, spec files, acceptance criteria, out-of-scope notes.
- Diff facts: changed files, churn, critical files, public APIs, dependencies, migrations.
- Test facts: pass/fail, added tests, changed tests, coverage delta if available.
- Security and architecture signals: auth, payment, permissions, secrets, eval/exec, shell, network, crypto, serialization, API shape.
- Harness telemetry: retries, tool calls, edit attempts, test runs, failed runs, errors, cost/tokens when available.

## Judgment discipline

- Do not judge only whether the code looks correct.
- Ask whether the PR did what was requested with the smallest reasonable change.
- Flag unrequested behavior changes and out-of-scope modifications.
- Distinguish missing evidence from proven safety.
- Prefer structured findings that can feed HarnessCI reports.

## Output shape

```text

  - ...

```
